# X (Twitter) Thread Repurposing Guide

This guide teaches agents how to transform long-form content (blogs, video transcripts) into high-performing X threads. Every framework, formula, and example below is derived from patterns observed in viral threads.

---

## Thread Architecture

Every thread follows one master structure:

```
Tweet 1 → HOOK (stop the scroll)
Tweet 2 → STRONGEST POINT (retain the reader)
Tweet 3-N → VALUE TWEETS (deliver insights)
Tweet N+1 → CTA (drive engagement)
```

### Optimal Thread Length
- **Sweet spot: 4–7 tweets**
- Long enough to deliver real value, short enough to hold attention
- Threads over 10 tweets see major drop-off unless the content is a gripping narrative
- **For repurposed content, target 5 tweets**

---

## Step-by-Step Repurposing Process

### Step 1: Extract the Skeleton

Read the source content and pull out exactly:
- **1 core thesis** → becomes the hook tweet
- **3–5 key supporting points** → become body tweets
- **1 main takeaway** → becomes the pre-CTA tweet

### Step 2: Rewrite for Spoken Tone

Threads that read like essays die. Rewrite in a conversational, first-person tone.

| ❌ Blog Tone | ✅ Thread Tone |
|:---|:---|
| "It is important to consider the implications of..." | "Here's the thing most people miss:" |
| "Research suggests that approximately 73% of..." | "73% of teams get this wrong." |
| "In conclusion, we can observe that..." | "The bottom line:" |
| "One should take into account..." | "Don't ignore this:" |

### Step 3: Front-Load Value

Put the most surprising or counterintuitive point in **Tweet 2** — right after the hook. This is where most readers drop off. If Tweet 2 delivers, they'll read the rest.

### Step 4: Apply the 1-3-1 Rule to Body Tweets

Inside each body tweet, structure the text as:
- **1 line**: Short, punchy opening sentence
- **3 lines**: 2–3 bullet points, facts, or examples
- **1 line**: Summary or bridge sentence leading to the next tweet

### Step 5: Cut Ruthlessly

- If a point doesn't add unique value, cut it
- A 5-tweet thread with 5 strong points beats a 10-tweet thread with 5 strong points and 5 filler
- Each tweet should deliver one standalone insight — valuable even if read alone

### Step 6: Keep the First Person Honest

Many hook formulas below read as lived experience ("I spent 6 months studying..."). **Never fabricate firsthand experience the poster doesn't have.** When repurposing someone else's article, the poster didn't do the studying or run the experiment — attribute it ("New breakdown of every viral tweet from 2024 — one pattern nobody talks about:") or frame it as discovery ("I just read a study of 40 apps. The numbers are uncomfortable:"). The specificity drives the engagement, not the pronoun — and one probing reply exposes a faked story. First-person formulas verbatim ONLY when the experience is genuinely the poster's.

## Hook Formulas

The first tweet determines everything. If the hook fails, nobody reads the thread. Use one of these proven formulas:

### 1. The Curiosity Gap
Create an information gap the reader must close.

```
Template: "I spent [time] studying [topic]. Here's what nobody talks about:"

Example: "I spent 6 months studying every viral tweet from 2024.
Here's the one pattern nobody talks about:"
```

### 2. The Contrarian / Hot Take
Challenge a widely held belief.

```
Template: "Unpopular opinion: [Common belief] is completely wrong. Here's why:"

Example: "Unpopular opinion: 'Consistency' is the worst advice for growing on X.
Here's why:"
```

### 3. The Listicle / Number Hook
Promise a specific, finite number of valuable items.

```
Template: "[Number] [things] that will [desirable outcome]:"

Example: "7 free AI tools that will save you 10+ hours a week:"
```

### 4. The Transformation / Before-After
Show a dramatic result.

```
Template: "[Time] ago, I was [bad state]. Today, I [good state]. Here's the [N]-step process:"

Example: "12 months ago, I had 200 followers.
Today, I have 50K.
Here's the 5-step process I followed:"
```

### 5. The "Most People" / "99% of People"
Creates an in-group/out-group dynamic.

```
Template: "99% of [people] don't know about [valuable thing]. Here's your unfair advantage:"

Example: "99% of founders don't know about this free legal resource.
Here's your unfair advantage:"
```

### 6. The "I Asked [Authority]"
Leverage borrowed credibility.

```
Template: "I asked a [expert] the one question everyone gets wrong about [topic].
Their answer surprised me:"
```

### Hook Rules
- **No "Thread 🧵" labels.** This looks spammy and reduces engagement.
- **Keep hooks under 200 characters** so they display fully without "Show more."
- **Start with a capital letter**, never a hashtag or @mention.
- **Use a line break after the hook** for visual breathing room.

## Body Tweet Formatting

### Use These Symbols for Bullets
```
→ Arrow points (preferred)
• Bullet dots
✅ Checkmarks (for lists of tips/do's)
❌ Crosses (for don'ts/mistakes)
```
Standard markdown dashes (`-`) do NOT render well on X.

### Line Breaks
- Each sentence or thought should be its own line
- Never write a wall of text in a single tweet
- Tweets with 2-3 short lines outperform dense paragraphs

### Emojis
- 1-2 per tweet maximum for visual anchoring (e.g., 🔑, 💡, ⚡)
- Use as layout guides, not decoration
- Never start a tweet with an emoji

### Numbering
- Always use "1/5", "2/5" format at the **start** of each tweet
- This tells readers the thread length and their position in it

## CTA Patterns (Last Tweet)

Dedicate the last tweet entirely to a CTA. Never introduce new information.

### Engagement CTA (Most Common)
```
If you found this valuable:

→ Like/RT the first tweet
→ Follow @handle for more [topic]
```

### Question CTA (Drives Comments)
```
Which of these surprised you the most?

Drop it below 👇
```

### Save/Bookmark CTA
```
Bookmark this thread for later.

I post threads like this every week on [topic].
Follow @handle so you don't miss the next one.
```

### Lead Magnet CTA
```
I turned this into a free [PDF/template/guide].

Like + Reply "SEND" and I'll DM it to you.
```

### Important: Links
- **Never put links in the first tweet** — this tanks reach
- Links in the **last tweet** are acceptable
- Better: post the link in a reply to the thread

## Complete Thread Examples

### Example 1: Repurposed from a "10 Productivity Tips" Blog

```
1/5
I tested 10 popular productivity methods for 30 days each.

Only 3 actually worked.

Here's what I found:

---

2/5
Method that worked #1: Time blocking (but with a twist)

→ Don't block tasks. Block energy types.
→ Deep work when fresh (morning)
→ Admin tasks during your afternoon slump
→ Creative work in the evening

Most people block by task type. Block by energy instead.

---

3/5
Method that worked #2: The 2-minute rule

→ If a task takes under 2 minutes, do it NOW
→ This cleared 40% of my daily to-do list
→ The mental relief is instant

The backlog of "small tasks" was draining more energy than the big ones.

---

4/5
Method that failed the hardest: Pomodoro (25 min on / 5 min off)

→ Great for shallow work
→ Terrible for deep work
→ The forced break at 25 min killed my flow state every time

If you're doing creative or technical work, try 90-minute blocks instead.

---

5/5
The #1 lesson from 10 months of testing:

No system works if you don't match it to YOUR energy patterns.

Stop copying other people's routines.
Build your own based on when you think best.

If this was useful, RT tweet 1 so others can find it 🔁
```

### Example 2: Repurposed from a YouTube Video Transcript (Dev Tutorial)

```
1/4
I just watched a mass migration from REST to GraphQL fail spectacularly.

Here's what went wrong (and what to do instead):

---

2/4
Mistake #1: They migrated everything at once

→ 200+ endpoints converted in a single sprint
→ No gradual rollout, no fallback
→ Production broke for 6 hours on launch day

The fix: Run REST and GraphQL in parallel. Migrate one service at a time.

---

3/4
Mistake #2: They didn't account for N+1 queries

→ GraphQL makes it easy to over-fetch without realizing
→ Their DB load tripled overnight
→ Dataloader wasn't implemented until week 3

Always implement batching/caching BEFORE you go live.

---

4/4
The takeaway:

GraphQL is powerful but it's NOT a drop-in replacement.

Treat it as a new architecture, not a format swap.

Found this useful? Follow for more engineering deep dives 👇
```

## Quality Checklist

Before outputting a thread, verify:

- [ ] Hook is under 200 characters and stops the scroll
- [ ] Each tweet is strictly under 280 characters
- [ ] Tweet 2 contains the strongest/most surprising point
- [ ] Every body tweet follows the 1-3-1 structure
- [ ] Thread uses "N/N" numbering format
- [ ] No "Thread 🧵" or "A thread:" labels anywhere
- [ ] Last tweet is a pure CTA with no new information
- [ ] Tone is conversational, not formal/academic
- [ ] No fabricated firsthand experience — first-person claims are genuinely the poster's, or attributed to the source
- [ ] Bullets use → or • instead of markdown dashes
- [ ] No links in tweet 1
- [ ] Thread is 4–7 tweets total
